# 调度器更新说明

## ✅ 数据清理功能已移除

根据您的需求，已从调度器中完全移除数据清理功能。

## 📝 更新内容

### 1. 移除的功能
- ❌ 数据清理任务 (`run_cleanup`)
- ❌ 清理检查 (`should_run_cleanup`)
- ❌ 日志清理 (`cleanup_logs`)
- ❌ 清理相关配置 (`CLEANUP_INTERVAL_DAYS`, `KEEP_DAYS`)

### 2. 保留的功能
- ✅ 数据采集（每天2次）
- ✅ 数据分析（每3天）
- ✅ 模型训练（每周）

## 📊 数据保留策略

**所有历史数据将永久保留，不会被删除。**

### 数据增长预估
```
当前数据量: 61,797 条（25天）
每月新增: ~75,000 条
每年新增: ~900,000 条
```

### 存储空间需求
```
每条记录: ~200 字节
每月数据: ~15 MB
每年数据: ~180 MB
```

**结论：存储需求很低，可以安全保留所有历史数据。**

## 🎯 当前调度策略

| 任务 | 频率 | 说明 |
|------|------|------|
| 数据采集 | 每天2次 | 08:00 和 20:00 |
| 数据分析 | 每3天 | 数据累积7500条时 |
| 模型训练 | 每周 | 周日自动训练 |

## 🚀 使用方式

### 运行调度器
```bash
# 一次性检查并运行到期任务
python scheduler.py --mode once

# 守护进程模式（持续监控）
python scheduler.py --mode daemon

# 查看当前状态
python scheduler.py --mode check
```

### 手动运行任务
```bash
# 数据采集
python run.py --mode collect

# 数据分析
python run.py --mode analyze

# 模型训练
python run.py --mode train
```

## 📈 监控建议

虽然不清理数据，但建议定期监控：

1. **磁盘空间** - 确保足够空间
2. **数据完整性** - 定期验证数据
3. **模型性能** - 监控预测准确度

