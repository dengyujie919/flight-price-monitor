### 1. 数据采集 📡

**频率**: 每天运行
**时间**: 建议 08:00 和 20:00（早晚各一次）
**原因**:
- ✅ 航班价格变化频繁
- ✅ 需要追踪价格趋势
- ✅ 您的系统已经稳定运行25天
- ✅ 每次采集只需约6-10秒

**配置**:
```python

```

**注意事项**:
- ⚠️ Amadeus API 测试版有调用限制
- ⚠️ 建议升级到付费版以获得更高配额
- ⚠️ 两次采集间隔建议≥6小时

---

### 2. 数据分析 📊

**频率**: 每3天运行一次
**原因**:
- 数据积累快（每天2,500条）
- 3天 = ~7,500条新数据
- 足以产生有意义的统计变化

**配置**:
```python
ANALYSIS_INTERVAL_DAYS = 3  # 从7天改为3天
```

**触发条件** (满足任一即可):
1. 距上次分析≥3天
2. 新增数据≥5,000条

---

### 3. 模型训练 🤖

**频率**: 每周运行一次
**原因**:
- 每周新增 ~17,500条数据
- 足以更新模型模式
- 平衡性能和计算成本

**配置**:
```python
TRAINING_INTERVAL_DAYS = 7  # 从14天改为7天
RETRAIN_THRESHOLD = 10000    # 提高到10,000条
```

**触发条件** (满足任一即可):
1. 距上次训练≥7天
2. 新增数据≥10,000条
3. 模型性能下降（MAE > 5元）

---

### 4. 数据清理 🧹

**频率**: 每月运行一次
**保留**: 最近90天的数据
**原因**:
- 90天数据足以训练模型
- 减少存储和计算负担
- 历史数据预测价值降低

**配置**:
```python
CLEANUP_INTERVAL_DAYS = 30
KEEP_DAYS = 90
```

## 🚀 自动化实现

### 方案一：使用 cron (Linux/Mac)

```bash
# 编辑 crontab
crontab -e

# 添加以下任务
0 8,20 * * * cd /path/to/predict && python run.py --mode collect
0 2 * * 0 cd /path/to/predict && python run.py --mode train
0 4 1 * * cd /path/to/predict && python run.py --mode cleanup
```

### 方案二：使用 Windows 任务计划程序

1. 打开"任务计划程序"
2. 创建基本任务
3. 设置触发器（每天/每周）
4. 操作：运行 `python run.py --mode [模式]`

### 方案三：使用 Python 定时任务

创建 `scheduler.py`:
```python
import schedule
import time
from run import run_collector, run_predictor

# 每天采集
schedule.every().day.at("08:00").do(run_collector)
schedule.every().day.at("20:00").do(run_collector)

# 每周训练
schedule.every().sunday.at("02:00").do(run_predictor)

while True:
    schedule.run_pending()
    time.sleep(60)
```

## 📊 数据量管理建议

### 存储空间预估
```
每条记录: ~200字节
每天数据: 2,500条 × 200字节 = 500KB
每月数据: 500KB × 30 = 15MB
三月数据: 15MB × 3 = 45MB
```

- 原始数据: 保留最近90天
- 特征数据: 保留最近60天

**压缩归档**:
- 超过90天的数据压缩为 `.zip`
- 每月归档一次

## ⚠️ 注意事项

### API 限制
- **测试版**: 有限制，建议每天≤2次
- **付费版**: 限制更高，可以更频繁采集
- **监控**: 记录API调用次数和错误率

### 性能优化
- 数据量>100万条时考虑分批训练
- 使用增量学习而非全量训练
- 定期清理日志文件

### 数据质量
- 每周检查数据完整性
- 验证价格异常值
- 检测重复数据
